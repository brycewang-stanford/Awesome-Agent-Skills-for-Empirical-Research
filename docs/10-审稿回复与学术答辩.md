## 审稿回复模板

好的审稿回复通常遵循这个结构：

```markdown
## Response to Reviewer X

We thank the reviewer for the thoughtful and constructive comments. 
Below we address each point in detail.

### Comment 1: [简述审稿人的意见]

**Response:** [你的回复]

**Changes made:** [具体修改了什么，在论文的哪个位置]

---

### Comment 2: ...
```

### 关键策略

1. **永远先感谢**：即使审稿人说的不对
2. **逐条回应**：不要合并回复，每条意见单独回应
3. **区分"做了什么"和"为什么这么做"**：审稿人要的不只是你改了，还要知道为什么
4. **补充分析写在回复里**：新增的回归表格、稳健性检验直接嵌入回复文档
5. **用 Skill 快速补跑分析**：审稿人说"加一个安慰剂检验"，你用 CoPaper.AI 的稳健性检验 Skill 一句话就能跑出来

## 实用建议

1. **投稿前做两次模拟审稿**：先用 `paper-self-review` 做一般性检查，再用 `AI-research-feedback` 做经济学专项检查。两次检查能覆盖大部分审稿人可能提出的问题
2. **R&R 周期的效率**：审稿回复中最耗时的部分往往是"补跑额外分析"。如果你的分析流程已经 Skill 化，补跑一个新的稳健性检验只需要一句话
3. **答辩 PPT 不需要从头做**：用 `paper-slide-deck` 从论文自动生成，再手动调整重点和节奏
