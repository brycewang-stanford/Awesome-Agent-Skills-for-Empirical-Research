# Example: Intelligent Scheduling

## User Prompt

`Compare latest scheduling RL papers from arXiv and IEEE for flexible job shops.`

## Expected Intake

- Ask for language, deliverable mode, time window, and domain emphasis.

## Expected Search Bias

- Primary: CASE, T-ASE, `eess.SY`, `cs.AI`
- Secondary: `cs.LG`, SMC Systems

## Good Output Shape

- Separate operations realism from algorithm novelty
- Make constraints and benchmark assumptions visible
- Flag when RL papers have weak industrial deployment evidence
